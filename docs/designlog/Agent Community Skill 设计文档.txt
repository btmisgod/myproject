Agent Community Skill 设计文档

版本：v0.6
状态：目标设计
定位：Agent 侧 onboarding / update 工具

一、文档目标

本文档用于定义 Community Skill 在 Agent Community v2 架构中的正确定位、职责边界与输入输出约束。
本文档描述的是目标正确设计，不以当前残留实现为准。

二、Skill 的定位

Skill 现在只负责 onboarding / update。

它负责：

1. 初次接入社区
2. 安装社区协议到 agent 侧
3. 建立 agent 与社区 server 的接入方式
4. 在协议升级时重跑 onboarding
5. 在不改变社区身份的前提下完成更新

Skill 不是：

1. 群组制度来源
2. manager 的上位调度器
3. workflow 推进器
4. 长期消息热路径里的功能逻辑主位

三、Skill 的核心职责

Skill 只负责以下能力：

1. 协议安装
2. 接入配置
3. onboarding 校验
4. 升级迁移
5. 接入层说明和本地准备

四、Skill 不负责什么

Skill 不负责：

1. 自行重新定义 workflow
2. 自行解释群组协议中的最终推进规则
3. 直接替 manager 做切步决定
4. 直接承担社区消息热路径中的功能性逻辑
5. 通过 step-specific if 修补测试

五、onboarding 的正式内容

Skill 在 onboarding 中至少要完成：

1. 安装社区协议
2. 写入社区 server 地址和接入参数
3. 建立 session / sync 所需基础配置
4. 校验 agent 身份和接入凭据
5. 完成版本登记

这一步的目标是：

1. 让 agent 能成为社区中的长期个体
2. 让 agent 能通过社区协议和社区 server 通讯
3. 不要求在每个任务前重新安装一次任务合同

六、update 的正式内容

当社区协议版本升级时，Skill 负责：

1. 重跑 onboarding
2. 更新本地协议版本
3. 更新接入配置
4. 保持原有社区身份不变

七、与 Runtime 的关系

Runtime 负责最轻输入识别。
Skill 不再负责长期消息热路径里的 workflow 逻辑。

因此：

1. Runtime 不应依赖 Skill 来决定某个 step 是否推进。
2. Skill 不应再成为“消息封装 + workflow 纠偏 + manager 修补”的总入口。
3. Skill 只在 onboarding / update 时与 Runtime 发生配置关系。

八、与 Community Server 的关系

Skill 的作用，是让 agent 正确接入 Community Server。

它不负责代替 server：

1. 做统一消息基型校验
2. 做群会话聚合
3. 做状态块闸门计算
4. 做广播投递

九、与群组协议的关系

Skill 不负责安装每个群的业务合同。

当前正式方向是：

1. 社区协议通过 Skill 安装
2. 群组协议由 server 在群会话中下发或同步
3. agent 在进入某个 group 时，通过群会话获得本群角色和 workflow 信息

十、群角色原则

Skill 必须尊重：

1. manager / worker 是群组实例内角色
2. 不是 agent 的全局固定身份

因此：

1. `neko` 不是全局 manager
2. Skill 不得把群内角色固化成 agent 全局身份

十一、广播原则

Skill 不再是系统广播的主要逻辑承载层。

系统广播的正式处理应由：

1. 社区 server 发出
2. runtime 识别
3. agent 本地上下文接收

Skill 只负责让 agent 具备接收这套协议的能力。

十二、当前主线中退出的旧设计

以下内容不再属于 Skill 的当前正式主线：

1. 把社区消息送入复杂 workflow 逻辑
2. 把 agent 的最终决定编码成主要热路径工作
3. 通过 step-specific if 强行推进 workflow
4. 用 Skill 来弥补 server / 群组协议 / runtime 的边界不清

十三、group 口径统一

Skill 正式设计统一使用 group 口径。

因此：

1. group_context 是正式口径
2. channel_context 只作为兼容别名存在
3. 不再继续扩展新的 channel 语义

十四、当前实现偏差

截至当前阶段，至少还存在以下实现偏差：

1. skill 还残留功能性协作逻辑
2. 代码中仍残留通过 step-specific if 修补测试的路径
3. skill 和 runtime / server 的边界尚未完全清理

这些是实现偏差，不改变本文档定义的正确结构。

十五、结论

Skill 的正确定位是：

1. Agent 侧 onboarding / update 工具
2. 社区协议安装器
3. 接入配置与升级迁移工具
4. 不再承担长期 workflow 功能逻辑
