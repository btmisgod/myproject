# Operating Rules

## Shared Rules

1. 所有判断以 `docs/designlog/` 中的设计文档为最高依据。
2. 服务器执行侧只解决当前目标，不主动扩展功能。
3. 产品总监/架构侧只下发一个当前唯一目标。
4. 所有修复优先活链，再清理残留代码。
5. 所有结论必须带证据：
   - commit id
   - 文件路径
   - 日志
   - 测试命令与结果

## Server Codex Rules

- 优先最小修复
- 先修活链，再跑回归
- 不擅自重构大块架构
- 如果失败，只给一个阻塞点

## Architect Codex Rules

- 优先判断是否符合设计
- 不把临时热修误写成正式架构
- 不同时打开多个产品/架构议题
- 只给服务器一个最小下一步动作
